C++

静态类型、编译类型、大小写敏感。

过程化编程、泛型编程。

学懂：

  1. 静态类型：

    1. 静态类型：编译时候就知道每一个变量的类型，因为类型错误而不能做的事情是语法错误
    2. 动态类型：编译时候不知道每一个变量的类型，因为类型错误而不能做的事情是运行时错误
    3. 强类型：不支持隐式类型变换。
    4. 弱类型：不支持隐式类型变换。
      • 显式类型变换：程序强制要求。

  2. 编译类型：

    1. 编译类型：将源代码编译成机器码，之后才能执行。
      • 优缺点：编译后成了机器码，可以脱离语言环境运行。编译以后如果需要修改，则需要整个代码重新编译
      • c、c++、swift
    2. 解释类型：运行时逐句翻译。
      • 优缺点；修改代码的时候可以直接修改，可以快速部署。每次都要解释，速度慢。
      • js、python、php
    3. 混合型语言
      • java：编译成为字节码（独立于平台），用jvm解释执行。

  3. 泛型编程：待解决。

    • https://blog.csdn.net/wue1206/article/details/81355251
    • 泛型：编译多态性。面向对象：运行多态性。

标准库：

  1. 核心语言，提供所有构件块.包括常量、数据类型和变量

  2. c++标准库，函数库，用于操作文件、字符串等

  3. 标准模板库（STL）（和前文提到的泛型有很大关系），提供大量方法，用于草屋哦数据结构

    • 与c++标准库的区别?

ANSI标准：规定了编译器允许的平台：MAC、unix、windows、alpha
